### iOS蓝牙


一、BLE 总体认知（先建立框架）

BLE 通信本质是：Central 通过 Characteristic 对 Peripheral 进行数据读写。

⭐ 架构分层（必须理解）

```

应用层（你写的协议：IMEI / WiFi / APN）
        ↓
GATT（服务/特征）
        ↓
ATT（数据传输协议）
        ↓
L2CAP
        ↓
PHY（蓝牙无线）


```

二、CoreBluetooth 核心对象

⭐ CoreBluetooth 核心角色

类                  作用
CBCentralManager    中央设备（手机）
CBPeripheral        外设
CBService           服务
CBCharacteristic    特征




1. CBCentralManager（中心设备）

手机端

```
职责：
    •   扫描设备
    •   连接设备
    •   管理连接状态
```

2. CBPeripheral（外设）

 蓝牙设备 比如（行车记录仪 / BLE模块）

 ```

 职责：
    •   提供服务（Service）
    •   提供特征（Characteristic）
```

3. CBService（服务）

功能模块

```
例：0F00 → UART服务

#define BluetoothUartServiceUUID        @"0F00" //服务UUID


//#define BluetoothSendcharacteristicUUID        @"00000F04-0000-1000-8000-00805F9B34FB" //写UUID
#define BluetoothReadIMEIcharacteristicUUID        @"0F03" //读imei

//#define BluetoothUartServiceUUID        @"00000F00-0000-1000-8000-00805F9B34FB" //服务UUID
#define BluetoothReadWriteWiFicharacteristicUUID        @"0F04" //读写Wi-Fi信息UUID

//#define BluetoothSetcharacteristicUUID        @"00000F00-0000-1000-8000-00805F9B34FB" //服务UUID
#define BluetoothReadWriteAPNcharacteristicUUID        @"0F05" //写UUID


//#define BluetoothReceivedcharacteristicUUID        @"00000F03-0000-1000-8000-008059634F6" //读UUID
#define PVMG10BluetoothServiceUUID        @"000F0E0D-0C0B-0A09-0807-060504030201"

```

```
 UUID 体系
    •   Service UUID → 功能模块
    •   Characteristic UUID → 数据通道
```

4. CBCharacteristic（特征）

 真正的数据通道（重点）

读 / 写 / 通知 都在这里发生

⭐ characteristic 三种操作

```

操作                API
读                 readValue
写                 writeValue
订阅               setNotifyValue

```

三、BLE 通信完整流程

初始化 CentralManager
        ↓
扫描 scanForPeripherals
        ↓
发现设备 didDiscover
        ↓
连接 connectPeripheral
        ↓
连接成功 didConnect
        ↓
发现服务 discoverServices
        ↓
发现特征 discoverCharacteristics
        ↓
读 / 写 / 订阅 notify
        ↓
接收数据 didUpdateValue
        ↓
断开连接

四、Characteristic 三大操作 （核心）

1. 读（Read）

```
[self.peripheral readValueForCharacteristic:characteristic];// 直接读取这个特征数据，会调用didUpdateValueForCharacteristic

```

```
 用于：

    •   IMEI
    •   状态查询
```

2. 写（Write）
```
[device.peripheral writeValue:data forCharacteristic:device.sendApnCharacteristic type:CBCharacteristicWriteWithResponse];

```

两种模式：
```
/*!
 *  @enum CBCharacteristicWriteType
 *
 *  @discussion Specifies which type of write is to be performed on a CBCharacteristic.
 *
 */
typedef NS_ENUM(NSInteger, CBCharacteristicWriteType) {
    CBCharacteristicWriteWithResponse = 0,
    CBCharacteristicWriteWithoutResponse,
};
```

```
WithResponse（可靠）
    •   有 ACK
    •   串行
    •   慢
WithoutResponse（高性能）
    •   无 ACK
    •   可并发
    •   快（推荐）

```

3. 通知（Notify）
```
 [self.peripheral setNotifyValue:YES forCharacteristic:characteristic.characteristic];
```
```
用于：
    •   实时数据
    •   主动推送
```

五、MTU（性能核心）



1.  定义
MTU = 单次最大传输单元

2.  关键公式
有效数据 = MTU - 3

3. 常见值

```
MTU       payload
23        20
185       182
247       244
```

数据传输限制
```
    •   默认 MTU ≈ 20 bytes
```

 iOS 特点
 
```
    •   自动协商
    •   不可手动设置
    •   必须动态获取：
```

```
NSUInteger sendLength = [self.bleLock.peripheral maximumWriteValueLengthForType:CBCharacteristicWriteWithoutResponse];

```
