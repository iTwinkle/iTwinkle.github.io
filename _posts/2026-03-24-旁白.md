### iOS 无障碍化(旁白适配)适配总结

```
要使用 VoiceOver 审核您的应用，您需要使用 VoiceOver 特有的手势来导航应用。测试时，您可以使用以下五个关键手势：

单指双击即可激活所选元素。
```

二、识别常见的无障碍问题

```
•	为应用元素添加辅助功能信息。默认情况下，VoiceOver 无法识别自定义 UI 元素。您需要为这些元素添加额外的辅助功能信息。
•	将元素分组，以便 VoiceOver 可以按正确的顺序浏览它们。VoiceOver的朗读顺序是从前到后。如果您希望 VoiceOver 以不同的顺序朗读元素，请使用分组功能，以便实现更符合您应用逻辑的导航。
•	添加描述性文本供 VoiceOver 朗读。依赖视觉提示的用户界面可能看起来不错，但对于 VoiceOver 用户来说可能难以使用。例如，VoiceOver 无法检测到用户选中确认按钮时按钮是否从灰色变为绿色。VoiceOver 可能只会描述元素，而不会描述其当前状态。请确保 VoiceOver 能够朗读按钮是否处于选中状态。

```

三、辅助功能标签和提示

1、对于 VoiceOver 无法访问的元素，首先要改进其辅助功能标签和提示。该属性提供描述性文本，当用户选择元素时，VoiceOver 会朗读这些文本；该属性则为所选元素提供额外的上下文（或操作）。


```
var accessibilityLabel: String? { get set }

```

讨论
标签是一个非常简短的本地化字符串，用于标识辅助功能元素，但不包含控件或视图的类型。例如，“保存”按钮的标签是“保存”，而不是“保存按钮”。

默认情况下，标准的 UIKit 控件和视图的标签源自其标题。但是，如果您提供的是自定义控件或视图，则需要正确设置此属性，以便辅助应用程序能够为残障用户提供准确的信息。

2、辅助功能标签非常重要，因为它们提供了 VoiceOver 朗读的文本。好的辅助功能标签应该简洁明了、信息丰富。需要注意的是，`<a>`UILabel和 `<an>`是不同的概念。默认情况下，VoiceOver 会朗读标准UIKit控件（例如 ` <a>`和 ` <a> `）的文本。但是，这些控件也可以拥有相应的属性，用于添加关于标签或按钮的更多详细信息。

4、使用身份检查器添加辅助功能标签和提示

有时，在 Xcode 中添加辅助功能标签和提示是不够的，例如当您使用 VoiceOver 无法自动识别的自定义 UI 元素，或者当您在辅助功能标签中使用变量时。在这些情况下，您需要以编程方式设置辅助功能标签或提示。您可以指定某个元素是辅助功能元素，然后创建相应的辅助功能标签和提示。

要使 VoiceOver 可以通过编程方式访问您的元素，请将其定义为辅助功能元素。

```
score.isAccessibilityElement = true
```

元素的标签在应用的整个生命周期内可能不会保持不变。例如，对于一个用于记录游戏分数的计数器，您可能需要随着分数的变化而更改其标签。这可以通过设置辅助功能标签和提示信息以编程方式实现。

```
score.accessibilityLabel = "score: \(currentScore)"
score.accessibilityHint = "Your current score" 
```

四、简化您的辅助功能信息

VoiceOver 会按照设备语言的顺序朗读。例如，VoiceOver 从左到右朗读英语，从右到左朗读阿拉伯语和波斯语。如果在用户界面中垂直堆叠标签，或在表格中显示文本，VoiceOver 可能无法按正确的顺序朗读标签。您可以以编程方式对辅助功能元素进行分组，以确保 VoiceOver 按预期朗读它们。例如，如果您正在创建一个应用，该应用堆叠标题和值来显示人员的姓名和电子邮件地址，那么根据它们在界面中的顺序，VoiceOver 可能无法同时朗读这些元素。请将这些元素分组，以确保创建清晰的上下文。

<img src="/images/posts/iOS 旁白适配/iOS 旁白适配2.png" > 

上图中，左侧有四个标签，VoiceOver 会从左到右依次朗读。虽然 VoiceOver 可以读取每个元素，但这并非最佳用户体验。右侧的标签分组后，VoiceOver 会按照预期的顺序朗读，从而实现清晰的导航。

要对标签进行分组，请创建一个标签，并添加要分组在一起的信息。

```
var elements = [UIAccessibilityElement]()
let groupedElement = UIAccessibilityElement(accessibilityContainer: self)
groupedElement.accessibilityLabel = "\(nameTitle.text!), \(nameValue.text!)"
groupedElement.accessibilityFrameInContainerSpace = nameTitle.frame.union(nameValue.frame)
elements.append(groupedElement)
```

五、旁白最常用 UIAccessibility API

```
•	accessibilityLabel： 描述内容“是什么”，例如 “产品名称”
•	accessibilityValue： 描述“值是多少”，例如“年化利率5%”
•	accessibilityTraits： 描述"特性、状态、行为、用法"。控件的特性可以是按钮(Button)，控件的状态可以是已选中(Selected)
•	accessibilityHint： 描述行动“做什么”，例如“点击两下可跳转购买页面”
```
1、Label、Value、Hint 配置的字符串，会以语音的形式念出来，配置什么，念什么。
2、 配置的Traits各个类型，也会以语音的形式念出。例如：UIAccessibilityTraitSelected、UIAccessibilityTraitImage、UIAccessibilityTraitButton分别播报"已选定”、”图片”、”按钮”。

4. 兼容开发者自定义的控件和视图,支持VoiceOver，自定义 View 组件无障碍化

所有的标准UIKit控件和视图默认支持VoiceOver，UILabel，UIButton这些控件的isAccessibilityElement属性默认是true, UIView属性默认是false，所以自定义视图如果想要支持VoiceOver，需要设置 isAccessibilityElement = YES, 使得VoiceOver可以访问这个view。

```
view.isAccessibilityElement = YES;
view.accessibilityLabel = @"文案内容";
view.accessibilityTraits = UIAccessibilityTraitNone;
```

5. 访问Cell与Cell中的按钮，且分别读出不同的内容

假设现在有-个UITableview与一个自定义Cell,，Cell中有一个按钮，现在需求是可以访问Cell与Cell中的按钮，且分别读出不同的内容。那么此时单纯设置Cell为访问元素然后再访问子元素这个思路是行不通的，我们可以设置CelI不激活为isAccessibilityElement, 但是苹果内部是可以将Cell作为整个访问元素且设置它的accessibilityL abel，完成button基本设置后， 再设置button的accessibilityHint就可以实现我们想要的效果。

```
- (UITableViewCell *)tableView:(UITableView *)tableView cellForRowAtIndexPath:(NSIndexPath *)indexPath {
    TestCell *cell = [tableView dequeueReusableCellWithIdentifier:[TestCell description] forIndexPath:indexPath];
    [cell setDataWithVersionM:self.dataArray[indexPath.row]];
    cell.isAccessibilityElement = NO;
    return cell;
}

.....
```

7. 数字/电话号连读处理，比如播报验证码1234的时候，系统会自动播报成“一千两百三十四”。

① 在中间加分隔符 1:2:3:4，这样就会一个数字一个数字的读了；
② 转换为对应的大写汉字即可；


self.cardNubLbl.accessibilityLabel = [YBVoiceOverTool acquireAccessableNumber:bankNum];

```
/// 将电话号码转换为旁白可读的中文
+ (NSString *)acquireAccessableNumber:(NSString *)number {
    NSArray *transFromArr = @[@"1",@"2",@"3",@"4",@"5",@"6",@"7",@"8",@"9",@"0"];
    NSArray *transToArr = @[@"一",@"二",@"三",@"四",@"五",@"六",@"七",@"八",@"九",@"零"];
    NSDictionary *dictionary = [NSDictionary dictionaryWithObjects:transToArr forKeys:transFromArr];
    NSMutableArray *resultArr = [NSMutableArray array];
    for (int i = 0; i < number.length; i++) {
        NSString *currentStr = [number substringWithRange:NSMakeRange(i, 1)];
        [resultArr addObject:dictionary[currentStr]?:currentStr];
    }
    NSString *resultStr = [resultArr componentsJoinedByString:@""];
    return resultStr;
}
```

9. 多音字读错

解决方案：换位同音字替代。比如：“密码重(chóng)置”，苹果识别为“密码重(zhòng)置”，可以用“密码崇置” 来解决。accessibilityLabel 属性仅仅旁白模式下朗读，不影响展示。

```
self.tipsLbl.accessibilityLabel = @“密码崇置”;
```

11. 一段文案中有两个跳转事件：点击隐私政策，只会跳转第一个，第二个一直跳转无效

解决方案：
a、改成两个控件，每个控件一个点击事件；
b、如果是富文本的话，直接用\n 换行，变为两行展示协议，可以分别跳转两个协议。

12. 支持卡片内容三指滑动

```
- (BOOL)accessibilityScroll:(UIAccessibilityScrollDirection)direction {
    if (direction == UIAccessibilityScrollDirectionLeft) {
        if (self.selectIndex < 4 ) {
            self.selectIndex = self.selectIndex+1;            
            [self scrollToIndex:self.selectIndex];
            [self accessibilityScrollPage:self.selectIndex];
            return YES;
        }
    }else if (direction == UIAccessibilityScrollDirectionRight) {
        if (self.selectIndex != 0) {
            self.selectIndex = self.selectIndex-1;
            [self scrollToIndex:self.selectIndex];
            [self accessibilityScrollPage:self.selectIndex];
            return YES;
        }
    }
    return NO;
}
```
13. 读取时间的无障碍

24 小时制的时刻系统无障碍会按字符一个个读出,需要转化成 12 小时制的写法,才会正常读取时间,例如 22:58 要写成 10.58PM.

14. UI 组件实际功能不符合时

使用 button 只做点击作用的时候(不需要选中态),需要设置对应的控件属性 accessibilityTraits

```
// 1. 不需要播报"已选中"
button.accessibilityTraits &= ~UIAccessibilityTraitSelected;
// 2. 纯文本
button.accessibilityTraits = UIAccessibilityTraitStaticText;

```
使用 laber / view / imageView, 实现自定义响应事件时,设置对应控件属性 accessibilityTraits 为 UIAccessibilityTraitButton.

15. hidden 元素

有时把某个 view 设成 hidden 的时候, UI 上已经不展示了, 但是VoiceOver仍然可以读到.

此时可以使用

```
UIAccessibilityPostNotification(UIAccessibilityLayoutChangedNotification, nil)

```

16. 弹窗

弹窗后屏蔽弹窗下面的元素

```
popupView.accessibilityViewIsModal = YES;//当前view才能响应无障碍播报
UIAccessibilityPostNotification(UIAccessibilityScreenChangedNotification, popupView.accessibilityElements.firstObject);
```

17. 焦点乱跳

焦点乱跳有两个可能

1、UICollectionView/UITableView reloadData 时候焦点会跳到最后一个 Cell, 解决办法为自行实现 cell 的更新, 减少直接 reloadData 调用; 如果这个 Cell 是满屏的, 可以设置当前 Cell 的 accessibilityViewIsModal 简单解决.
2、某个 UI 元素被加入到 accessibilityElements 但是又被标记为不可用 isAccessibilityElement = NO, 在手指左划时会出现焦点乱跳(但是右划正常)

18. Cell 横划删除 在无障碍下无法使用

使用自定义操作来代替

```
UIAccessibilityCustomAction * action = [[UIAccessibilityCustomAction alloc] initWithName:@"测试" target:self selector:@selector(testToast)];
UIAccessibilityCustomAction * action1 = [[UIAccessibilityCustomAction alloc] initWithName:@"测试1" target:self selector:@selector(testToast1)];
UIAccessibilityCustomAction * action2 = [[UIAccessibilityCustomAction alloc] initWithName:@"测试2" target:self selector:@selector(testToast2)];

self.accessibilityCustomActions = @[action, action1, action2];

```

触发方法:

1、单击选中该 UI 元素
2、单指上下划, 第一次播报 “测试”, 再单指上下划第二次播报 “测试2”, 继续划会有第三次播报 “测试3”, 第四次播报”激活” (初始状态).
3、然后在某一次播报后单指双击, 就可以触发对应的 selector

19. 需要播报”已选中”

```
// 设置"已选中"播报
view.accessibilityTraits |= UIAccessibilityTraitSelected;
// 移除"已选中"播报
view.accessibilityTraits &= ~UIAccessibilityTraitSelected;
```
